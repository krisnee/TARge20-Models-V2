Extreme programming model: Advantages and Disadvantages

Advantages Disadvantages
➕ Extreme programming model allows software development companies to save costs and time required for project realization. ➖ Extreme programming model is focused on the code rather than on design.
The developers who prefer to use this methodology create extremely simple code that can be improved at any moment. This methodology does not measure code quality assurance.
The whole process in XP is visible and accountable. XP is not the best option if programmers are separated geographically.
